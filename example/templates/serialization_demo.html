Example 5: Using Serialization in Python Code

You can also serialize menus in your Python views for API responses:

from flex_menu import root, serialize_menu def menu_api_view(request): # Get and process the menu menu = root.get('main_navigation') processed = menu.process(request) # Serialize to JSON json_data = serialize_menu(processed, indent=2) # Return as API response return JsonResponse(json.loads(json_data), safe=False) # Or use the dictionary directly def menu_data_view(request): menu = root.get('main_navigation') processed = menu.process(request) # Get dictionary representation menu_dict = processed.to_dict() return JsonResponse(menu_dict)

Use Cases for Menu Serialization

  • Responsive Menus: Render different menu layouts for mobile vs desktop using the same data
  • JavaScript Frameworks: Use with Vue.js, React, Alpine.js, or vanilla JavaScript
  • API Endpoints: Expose menu structure via REST APIs for SPAs or mobile apps
  • Client-Side Filtering: Let users search/filter menu items without page reloads
  • Progressive Enhancement: Start with server-rendered HTML, enhance with JavaScript
  • A/B Testing: Dynamically render different menu experiences

Available Menu Item Properties

Each serialized menu item includes these properties:

{
  "name": "item_name",              // Unique identifier
  "url": "/path/to/page/" or null,  // Resolved URL
  "visible": true,                  // Visibility after checks
  "selected": false,                // True if URL matches current page
  "depth": 0,                       // Depth in menu tree (0 = root)
  "has_children": false,            // Has child menu items
  "has_visible_children": false,    // Has visible child items after processing
  "is_clickable": true,             // Has a URL (clickable link)
  "extra_context": {                // Custom data from menu definition
    "label": "Display Name",
    "icon": "fas fa-home",
    "custom_field": "any value"
  },
  "children": []                    // Array of child menu items
}
💡 Pro Tip: All serialization features maintain backward compatibility. You can continue using traditional server-side rendering alongside serialized menus in the same project!
